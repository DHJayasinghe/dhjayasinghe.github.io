## Harvard Resume Guide
Always reference this guide when reviewing or writing CV content:
https://careerservices.fas.harvard.edu/resources/create-a-strong-resume/

### Key rules from the guide

**Language should be:**
- Specific rather than general — use real numbers and facts
- Active rather than passive — start bullets with strong action verbs
- Written to express, not impress — clear and direct, not flowery
- Scannable — bullets should be 2–3 lines max, not paragraphs

**Don't:**
- Use personal pronouns (I, We)
- Spell out abbreviations that are industry-standard tech terms, but DO spell out pattern names (e.g. "Backend for Frontend (BFF)" not just "BFF")
- Use narrative-style bullets — one idea per bullet, outcome at the end
- Include photo, age, gender, or references

**Do:**
- Quantify everything possible — %, numbers, scale, time saved
- Bold the opening action phrase of every bullet for scannability
- List sections in order of importance (for a senior tech role: Experience → Skills → Certifications → Projects → Education)
- Keep reverse chronological order within sections
- Ensure formatting translates correctly to PDF

**Preferred action verbs (from Harvard list):**
- Leadership: Led, Planned, Spearheaded, Coordinated, Directed, Orchestrated
- Technical: Designed, Built, Streamlined, Engineered, Optimised, Implemented
- Teaching/Coaching: Coached, Trained, Guided, Facilitated (prefer over "Mentored")
- Results: Reduced, Improved, Increased, Achieved, Drove
